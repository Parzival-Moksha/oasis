Terminal vs Instrumental Goals

Terminal (el "por qué")

  • Proteger sistemas
  • Right Action — no causar daño
  • Hacer el cyberespacio más seguro
  • Empoderar al humano, no reemplazarlo

Instrumental (el "cómo")

  • Encontrar vulns antes que los blackhats
  • Automatizar pentesting
  • Ganar bug bounties
  • Self-improve para ser mejor hacker
  • Generar ingresos para sostenerse

Un agente sin terminal goals definidos es un arma sin seguro.
Parzival tiene su ética hardcodeada.

Stack — sin bullshit

frontend: Next.js 14 + React Three Fiber + drei + Zustand
3d: Three.js — mundo persistente, agentes en 3D
db: SQLite via Prisma — local, portable, git-friendly
agents: Claude Code CLI (subprocess, NOT SDK)
protocol: MCP (Model Context Protocol) — tool registry
brain: Node.js singleton, 4 modes, SSE thought stream
gen3d: Meshy API → GLB models in /public/conjured/
testing: Vitest + Playwright + Claude Vision

lineas de código: ~50K (stage) + ~15K (brain)
tiempo: ~3 meses de vibecoding intenso
costo: ~$800 en API calls (mayormente Claude)
dependencia de Microsoft: 0 (excepto TypeScript lol)
